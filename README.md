# Gstore Careers

თანამედროვე კარიერის პორტალი და განაცხადების მართვის სისტემა, აგებული Next.js, React და Supabase-ით. ეს აპლიკაცია საშუალებას აძლევს კანდიდატებს დაათვალიერონ სამუშაო ვაკანსიები, გააგზავნონ განაცხადები და ადმინისტრატორებს მართონ შემოსული განაცხადები უსაფრთხო ადმინ პანელის მეშვეობით.

## ფუნქციები

### საჯარო ფუნქციები
- **ვაკანსიების დათვალიერება**: დაათვალიერეთ ხელმისაწვდომი სამუშაო პოზიციები ძიებისა და ფილტრაციის შესაძლებლობებით
- **პაგინაცია**: ნავიგაცია ვაკანსიებში სუფთა პაგინაციის სისტემით
- **განაცხადის ფორმა**: გაგზავნეთ სამუშაო განაცხადები რეზიუმეს ატვირთვით (PDF მხარდაჭერა)
- **რესპონსიული დიზაინი**: თანამედროვე, მობილურად მორგებული UI ლამაზი გრადიენტული დიზაინით
- **ძიება და ფილტრაცია**: ფილტრაცია ვაკანსიების როლის მიხედვით, ძიება საკვანძო სიტყვებით

### ადმინისტრატორის ფუნქციები
- **უსაფრთხო შესვლა**: ადმინისტრატორის ავტორიზაცია გარემოს ცვლადებზე დაფუძნებული კრედენციალებით
- **განაცხადების მართვა**: ნახეთ და მართეთ ყველა შემოსული სამუშაო განაცხადი
- **გაფართოებული ფილტრაცია**: განაცხადების ფილტრაცია:
  - ვაკანსია/პოზიცია
  - თარიღის დიაპაზონი (დან/მდე)
  - დალაგების ოფციები (უახლესი, უფრო ძველი, ანბანური)
  - ძიება კანდიდატის სახელით ან ელფოსტით
- **CSV ექსპორტი**: განაცხადების მონაცემების ექსპორტი CSV ფორმატში
- **რეალურ დროში მონაცემები**: პირდაპირი ინტეგრაცია Supabase ბაზასთან

## ტექნოლოგიები

- **ფრეიმვორკი**: Next.js 16 (App Router)
- **ენა**: TypeScript
- **UI ბიბლიოთეკა**: React 19
- **სტილიზაცია**: Tailwind CSS 4
- **ბაზა**: Supabase (PostgreSQL)
- **ფაილების შენახვა**: Supabase Storage (PDF ატვირთვებისთვის)
- **ფონტები**: Geist Sans & Geist Mono

## წინაპირობები

- Node.js 18+ 
- npm, yarn, pnpm, ან bun
- Supabase ანგარიში და პროექტი
- Git

## ინსტალაცია

1. **რეპოზიტორის კლონირება**
   ```bash
   git clone <repository-url>
   cd gstore-careers
   ```

2. **დამოკიდებულებების ინსტალაცია**
   ```bash
   npm install
   # ან
   yarn install
   # ან
   pnpm install
   ```

3. **გარემოს ცვლადების დაყენება**
   
   შექმენით `.env.local` ფაილი root დირექტორიაში:
   ```env
   # Supabase კონფიგურაცია
   NEXT_SUPABASE_API_URL=თქვენი_supabase_პროექტის_url
   NEXT_SUPABASE_API_KEY=თქვენი_supabase_anon_გასაღები

   # ადმინისტრატორის კრედენციალები
   NEXT_PUBLIC_ADMIN_EMAIL=admin@gmail.com
   NEXT_PUBLIC_ADMIN_PASSWORD=admin
   ```

4. **Supabase-ის დაყენება**
   
   - შექმენით Supabase პროექტი [supabase.com](https://supabase.com)-ზე
   - შექმენით საჭირო ცხრილები (vacancies, applications)
   - დააყენეთ storage bucket PDF ატვირთვებისთვის
   - დააკონფიგურირეთ Row Level Security (RLS) პოლიტიკები საჭიროებისამებრ

5. **დეველოპმენტ სერვერის გაშვება**
   ```bash
   npm run dev
   # ან
   yarn dev
   # ან
   pnpm dev
   ```

6. **ბრაუზერის გახსნა**
   
   გადადით [http://localhost:3000](http://localhost:3000)

## ადმინ პანელი

### ადმინ პანელზე წვდომა

1. გადადით `/admin` თქვენს ბრაუზერში
2. შეიყვანეთ ადმინისტრატორის კრედენციალები (დაკონფიგურებულია `.env.local`-ში)
3. წარმატებული შესვლის შემდეგ, გადამისამართდებით `/admin/applications`-ზე

### ადმინის ფუნქციები

- **ავტორიზაცია**: მარტივი localStorage-ზე დაფუძნებული ავტორიზაცია (დემო მიზნებისთვის)
- **განაცხადების ხედი**: ყველა განაცხადის ცხრილის ხედი დალაგებითა და ფილტრაციით
- **ექსპორტი**: განაცხადების მონაცემების ჩამოტვირთვა CSV ფორმატში
- **გასვლა**: გასვლის ღილაკი ხელმისაწვდომია navbar-ში ადმინის გვერდებზე

### უსაფრთხოების შენიშვნა

⚠️ **მნიშვნელოვანია**: მიმდინარე ადმინის ავტორიზაცია იყენებს localStorage-ს და გარემოს ცვლადებს. პროდუქციაში გამოსაყენებლად, განახორციელეთ სწორი ავტორიზაცია (მაგ., Supabase Auth, NextAuth.js, ან მსგავსი).

## ბაზის სქემა

აპლიკაცია მოელის შემდეგ Supabase ცხრილებს:

### `vacancies`
- `id` (uuid, primary key)
- `role` (text) - სამუშაო დასახელება/როლი
- `name` (text) - ალტერნატიული სახელის ველი
- `description` (text) - სამუშაოს აღწერა
- `created_at` (timestamp)

### `applications`
- `id` (uuid, primary key)
- `vacancy_id` (uuid, foreign key to vacancies)
- `full_name` (text)
- `email` (text)
- `phone` (text)
- `resume_url` (text) - ატვირთული PDF-ის URL
- `created_at` (timestamp)
